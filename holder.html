1. Easy to navigate
A user interface with a navigation flow which mimics that which a user would naturally/comfortably make is the best type of page, as it improves the overall user experience
2. Quick loading time
Fast loading times improve the ease of navigation and user involvement with the website
3. Interactive
The user interface is responsive and uses web tools to make components of the page responsive to specific user interactions, in order to make it more intuitive
4.Intuitive
A site should be easy to navigate through without additional description. Every feature and component should be designed to make sense to every user.
5. Simple
A simple page that is void of clutter is easier to navigate and will result in an elevated user experience on the site. ‘100 simple pages > 1 cluttered one’.

Web accessibility uses varying inclusive practices to ensure a web page is free from barriers that could prevent people with certain disabilities or restrictions from interacting with the site.

Some of these include:

Visual: partial vision or the inability to perceive colour or contrast
Hearing: Reduced hearing ability
Motor/physical: Mobility issues, difficulty moving certain body parts or fine motor movement
Photosensitive seizures: conditions such as epilepsy can be triggered by flashing lights and cause seizures
Cognitive: conditions such as dementia and dyslexia may affect cognitive ability
Socio-economic: certain socioeconomic factors can cause restrictions on bandwidth and speed of internet when browsing

Considerations:

Colour: Due to visual impairments, colour cannot be solely used to covey information, but other design considerations should be used to ensure the visually impaired have the same intuitive experience. Additionallt, colours which have a distinct contrast should be used when overlaying colours.
Structure: Arranging the contents of the webpage will improve the flow for Users when using the site- especially those with cognitive difficulties.
Alt text: the use of alt text for images allows a description of photographs to be read out to users who wouldn’t otherwise have access to the image

GDPR

Definition:

GDPR stands for 'General Data Protection Regulation' and is a legal data protection and privacy framework upheld by the European Union which defines how personal information of EU residents is processed, and can be seen as an expansion of the Data Protection Act 1998. EU residents have autonomy over their data and can request a copy of any of their stored personal data. They can also request to be “forgotten” by entities that hold their data.
EU Residents have the following rights under GDPR, therefore the following must be upheld:
1. Right to be informed
Be transparent about the intended use of personal data. Typically handled through the site’s privacy policy.
2. Right of access
Data must be provided on request, and should be provided in commonly used format such as CSV.
3. Right to rectification
Allow a client to correct incomplete or inaccurate information.
4. Right to erasure
Clients can request deletion or removal of personal data when there is no compelling reason for its continued processing. Also referred to as “the right to be forgotten.”
5. Right to restrict processing
Data can be stored but no longer processed if clients block the processing of their personal data.
6. Right to portability
Allow individuals to obtain and reuse their personal data for their own purposes.
7. Right to object
Individuals can object to having their personal information used. This includes for purposes of direct marketing, research and statistics.
8. Rights related to automatic decision making, including profiling
This rule specifies when you can use profiling and automated decision making. It also defines requirements that must be met, such as the individual providing explicit consent.
